## 🦾 什么是聚合兽？

聚合兽是一个**智能 API 聚合网关**，解决以下痛点：

- ❌ 单个 API 挂了，整个服务崩溃
- ❌ 需要管理多个 API Key
- ❌ 不同 API 返回格式不统一
- ❌ 不知道哪个数据源更可靠

**聚合兽的解决方案**：
- ✅ 多数据源自动聚合
- ✅ 主源挂了自动切备用
- ✅ 统一返回格式
- ✅ 置信度评分 (0-1)

## 📊 响应格式

### 标准字段

所有聚合响应包含以下字段：

```json
{
  "success": true,              // 请求是否成功
  "data": { ... },              // 实际数据
  "source": "https://...",      // 数据来源 URL
  "confidence": 0.95,           // 可信度 (0-1)
  "latency_ms": 211,            // 响应延迟 (毫秒)
  "timestamp": "ISO8601",       // 时间戳
  "fallback_used": false,       // 是否使用了降级数据
  "remaining_requests": -1      // 剩余请求配额 (-1=无限)
}
```

### 置信度说明

| 置信度 | 含义 | 场景 |
|--------|------|------|
| **0.9-1.0** | 极高 | 主数据源，低延迟 |
| **0.7-0.9** | 高 | 备用数据源 |
| **0.5-0.7** | 中 | 降级数据 |
| **<0.5** | 低 | 估算值/缓存 |

## 🔄 自动降级策略

### 工作原理

```
用户请求
   ↓
尝试主数据源 (priority=1)
   ↓ 失败？
尝试备用源 1 (priority=2)
   ↓ 失败？
尝试备用源 2 (priority=3)
   ↓ 失败？
返回估算值 (fallback)
```

### 示例：黄金价格

**配置**：
```python
"gold": [
  DataSource(url="metals.live", priority=1),  # 主源
  DataSource(url="data:...", priority=2),     # 估算值 fallback
]
```

**场景 1：主源正常**
```json
{
  "data": {"price": 2680},
  "source": "https://api.metals.live/...",
  "confidence": 0.92,
  "fallback_used": false
}
```

**场景 2：主源失败，降级**
```json
{
  "data": {"price": 2650, "fallback": true},
  "source": "data:text/plain,...",
  "confidence": 1.0,
  "fallback_used": true
}
```

## 🛠️ 使用场景

### 场景 1：市场追踪器

```python
import requests

API_KEY = "juheshou_your_key"
BASE_URL = "http://localhost:8001"

def get_market_report():
    btc = requests.get(f"{BASE_URL}/v1/aggregate/btc", 
                       headers={"Authorization": f"Bearer {API_KEY}"}).json()
    eth = requests.get(f"{BASE_URL}/v1/aggregate/eth", 
                       headers={"Authorization": f"Bearer {API_KEY}"}).json()
    gold = requests.get(f"{BASE_URL}/v1/aggregate/gold", 
                        headers={"Authorization": f"Bearer {API_KEY}"}).json()
    
    return {
        "btc_price": btc["data"]["price"],
        "eth_price": eth["data"]["price"],
        "gold_price": gold["data"]["price"],
        "timestamp": btc["timestamp"]
    }
```

### 场景 2：监控告警

```python
def check_price_alert():
    btc = requests.get(f"{BASE_URL}/v1/aggregate/btc", 
                       headers={"Authorization": f"Bearer {API_KEY}"}).json()
    
    if btc["data"]["price"] > 70000:
        send_alert("BTC 突破 $70,000!")
    
    if btc["confidence"] < 0.7:
        send_alert("⚠️ 数据可信度低，检查数据源")
```

### 场景 3：数据对比

```python
# 获取多个数据源对比
sources = requests.get(f"{BASE_URL}/v1/sources", 
                       headers={"Authorization": f"Bearer {API_KEY}"}).json()

for source in sources["sources"]:
    print(f"{source['name']}: 优先级 {source['priority']}, "
          f"成功率 {source['success_rate']:.2%}")
```

## 🐛 错误处理

### 常见错误

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| `Missing API Key` | 未提供认证头 | 添加 `Authorization: Bearer ...` |
| `Invalid API Key` | Token 无效 | 检查 API Key 是否正确 |
| `未找到数据源: xxx` | 端点不存在 | 检查端点名称（见 API 端点列表） |
| `所有数据源失败` | 所有数据源不可用 | 稍后重试，联系管理员 |

### 错误响应格式

```json
{
  "detail": "错误描述信息"
}
```

## 🎯 最佳实践

### 1. 检查置信度

```python
response = requests.get(url, headers=headers).json()
if response["confidence"] < 0.7:
    print("⚠️ 数据可信度低")
```

### 2. 处理降级数据

```python
if response["data"].get("fallback"):
    print("⚠️ 使用估算值，非实时数据")
```

### 3. 缓存优化

```python
# 聚合兽内置缓存 (TTL=60 秒)
# 60 秒内重复请求直接返回缓存
for i in range(10):
    response = requests.get(url)  # 只有第一次会请求外部 API
```

### 4. 错误重试

```python
import time

for attempt in range(3):
    response = requests.get(url)
    if response.status_code == 200:
        break
    time.sleep(2 ** attempt)  # 指数退避
```
